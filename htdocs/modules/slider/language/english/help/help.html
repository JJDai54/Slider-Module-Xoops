Help: slider - Slides management

Description

This module allows you to manage ; the slides of the home page of the themes used
It only works with the "xbootstrap" type themes.
The file " tpl / slider .tpl "must exist.
It may be necessary to write to it to allow modification.

Functionality < / strong>:

  • Adding, Removing slides & agrave; the home page
  • Activation, deactivation of slides, without deletion
  • Defines the type of display: with or without display period
  • Defining the display order of slides
  • Defining a display period to anticipate an event


Installation / Uninstallation

No special measures are necessary, follow the installation process standard and extract the cursor folder to the ../ modules directory. Install the module through Admin => System module => Modules.
Detailed instructions on installing modules are available in XOOPS Operations Manual

 

Features

Tutorial

Once installed, actions to perform:

  • Standard installation of module
  • Add slides to the themes used (see "Adding a new slide)" "
  • Activate the module block, regardless of its place it will remain invisible
    Important : the "Show block" option of the block Allows you to list the selected slides
    leave it on "No" in production

Adding a new slide

Short name (sld_short_name) : Phrase for display in admin. Used for sorts, but not used for display on slides.

Title (sld_title) : message displayed on the slide. With TinyMCE it is possible to make the title more visible on certain slides.

Description (sld_description) : Message displayed under the title on the slide. With TinyMCE it is possible to make the description more visible on certain slides.

Read more (sld_read_more) : URL accessible on the button displayed on the slide. Optional, it provides access to an article or an information page.

Image (sld_image) : Image of slide. Please check the size of the files (1920 px x 500 px for xbootstrap).

Active (sld_actif) : no = The slide will never be displayed. Allows you to activate a slide for later use without deleting it, for example for recurring holidays.

Visible (sld_always_visible) : yes = the slide is always displayed regardless of the display period / no = the display period must be defined.

Start date (sld_date_begin) : start date and time of slide display.

End date (sld_date_end) : end date of slide display.

Weight (sld_weight) : defines the display order of the slides. If several slides have the same weight, the alphabetical order of the titles is applied.

Theme (sld_theme) : Name of the theme on which the slides will be displayed. The "tpl / slides.tpl" file will be modified.

Treatments

On the index page several treatments are   possible.

  • Resets the original 'tpl / slider.tpl' files for each theme.
    Disable the slider module block to allow display of original slides
    Remove caches to refresh display immediately

 

  • Activate the block that allows the setting a day of slides
    Equivalent to display the block in the block manager

 

Block => slider_update_tpl

When the block is activated, the block modifies the "tpl/slider.tpl" file with the slides that have been used.

The setting day is done that in the case one of the slides has been modified added or deleted.
The original file is saved; with the suffix "-old" => "tlp / slider-old.tpl"
It will be re-installed; when uninstalling the module.
