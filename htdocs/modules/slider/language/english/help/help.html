Help: slider - Slides management

Description

This module allows you to manage ; rer the slides of the home page of the themes used
It only works with the "xbootstrap" type themes.
The file " tpl / slider .tpl "must exist.
It may be necessary to write to it to allow modification.

Functionality < / strong>:

  • Adding, Removing slides & agrave; the home page
  • Activation, deactivation of slides, without deletion
  • Defines the type of display: with or without display period
  • Defining the display order of slides
  • Defining a display period to anticipate an event


Installation / Uninstallation

No special measures are necessary, follow the installation process standard and extract the cursor folder to the ../ modules directory. Install the module through Admin - & gt; System module - & gt; Modules.
Detailed instructions on installing modules are available in XOOPS Operations Manual

 

Features

The module "slider" continues to evolve. For this reason, I invite all developers to report and send to the SVN any changes or additions in this module, so that we can jointly contribute to the development

Tutorial

Once installed, actions to perform:

  • Standard installation of module
  • Add slides to the themes used (see "Adding a new slide)" "
  • Activate the module block, regardless of its place it will remain invisible
    Important : the "Show block" option of the block Allows you to list the selected slides
    leave it on "No" in production

Adding a new slide

Title: message displayed on the slide .
Description: Message displayed under the title on the slide.
Image: Image of the slide. Please check the file size (1920px x 500px for xbootstrap).
Active : no = The slide will never be displayed. Allows you to activate a slide for later use without deleting it er, for example for recurring holidays.
Visible: yes = the slide is always displayed; -  no = the display period must be defined.
Start date: date and time when the slide starts to display.
End date: end date of display of the slide.
Weight: defines the order in which the slides are displayed. If several slides have the same weight the alphabetical order of the titles is applied.

Treatments

On the index page several treatments are   possible.

  • Resets the original 'tpl / slider.tpl' files for each theme.
    Disable the slider module block to allow display of original slides
    Remove caches to refresh display immediately

 

  • Activate the block that allows the setting a day of slides
    Equivalent to display the block in the block manager

 

Block - > slider_update_tpl

When the block is activated, the block modifies the "tpl/slider.tpl" file with the slides that have been used.

The setting day is done that in the case one of the slides has been modified added or deleted.
The original file is saved; with the suffix "-old" -> "tlp / slider-old.tpl"
It will be re-installed; when uninstalling the module.
