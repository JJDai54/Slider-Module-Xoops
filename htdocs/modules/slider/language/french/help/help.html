Help: slider - Slides management

Description

Ce module permet de gérer les slides de la page d'accueil des thèmes utilisés
Il ne fonctionne qu'avec les thèmes de type "xbootstrap".
Le fichier "tpl/slider.tpl" doit exister.
Il sera peut être nécéssaire de la mettre en écriture pour permettre sa modification.

Fonctionalités :

  • Ajout, Supprimer de slides à la page d'accueil
  • Activation, désactivation des slide, sans supression
  • Défini le type d'affichage: avec ou sans période d'affichage
  • Définition de l'ordre d'affichage des slides
  • Définition d'une période d'affichage pour anticiper un événnement

 

Install/uninstall

Aucune mesure particulière n'est nécessaire, suivez le processus d'installation standard et extrayez le dossier du curseur dans le répertoire ../modules. Installez le module via Admin => Module système => Modules.
Des instructions détaillées sur l'installation des modules sont disponibles dans le XOOPS Operations Manual

 

Caratéristiques

Le module « slider » continue d’évoluer. Pour cette raison, j'invite tous les développeurs à signaler et envoyer sur le SVN tout changement ou ajout à ce module, afin que nous puissions contribuer conjointement au développement

Tutorial

Une fois installé, actions à effectuer :

  • Installation standard  de module
  • Ajouter des slides au thèmes utilisés (voir "Ajout d'un nouveau slide)""
  • Activer le block du module, peu importe sa place il restera invisible
    Important : l'option "Afficher le block" du block Permet de lister les slides sélectionnés
                  à utiliser en mode débogage uniquement
                  laisser la sur "Non" en production

Ajout d'un nouveau slide

Nom court (sld_short_name): Expression pour affichage dans l’admin. Utilisé pour les tris, mais inutilisé pour l’affichage sur les slides.

Titre (sld_title) : message affiché sur le slide. Avec TinyMCE il est possible de rendre le titre plus visible sur certains slide.

Description (sld_description) : Message affiché sous le titre sur le slide. Avec TinyMCE il est possible de rendre la description plus visible sur certains slide.

En savoir plus (sld_read_more) : URL accessible sur le bouton affiché sur le slide. Optionnelle, il permet d’accéder à un article ou une page d’informations.

Image (sld_image) : Image du slide. Attention vérifiez la taille des fichiers (1920 px x 500 px pour xbootstrap).

Actif (sld_actif) : non = Le slide ne sera jamais afficher. Permet d’activer un slide pour une utilisation ultérieure sans le supprimer, par exemple pour les fêtes récurrentes.

A une période (sld_periodicity) : non = le slide est toujours affiché quel que soit la période d’affichage /  oui = il faut définir la période d'affichage.

Date de début (sld_date_begin) : date et heure de début d'affichage du slide.

Date de fin (sld_date_end) : date de fin d'affichage du slide.

Poids (sld_weight) : défini l'ordre d'affichage des slides. Si plusieurs slides ont le même poids l'ordre alphabétique des titres est appliqué.

Thème (sld_theme) : Nom du thème sur lequel les slide seront affichés. Le fichier "tpl/slides.tpl" sera modifié.

Traitements

Sur la page d'index plusieurs traitements sont  possibles.

  • Réinitialise les fichiers 'tpl/slider.tpl' d'origine de chaque thème.
    Désactive le block du module slider pour permettre l'affichage des slides d'origine
    Supprime les caches pour actualiser l'affichage immédiatement

 

  • Activer le block qui permet la mise à jour des slides
    Equivaut à afficher le block dans le gestionnaire de blocks

 

Block => slider_update_tpl

Quand le block est activé, le block modifie le fichier "tpl/slider.tpl" avec les slides qui ont été ajoutés.

La mise à jour n'est effectué que dans le cas un des slides a été modifié ajouté ou supprimé.
Le fichier d'origine est sauvegardé avec le le suffixe "-old" => "tlp/slider-old.tpl"
Il sera réinstallé lors de la désinstallation du module.
